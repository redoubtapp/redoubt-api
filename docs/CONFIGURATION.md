# Configuration Reference

Redoubt is configured through three mechanisms: environment variables in `.env`, application settings in `config/config.yaml`, and service-specific files for LiveKit and Caddy.

## Environment Variables (.env)

The `.env` file is located at `/opt/redoubt/.env` and is read by Docker Compose. It provides secrets and deployment-specific values that are interpolated into the config files and passed to containers.

| Variable | Description | Required | Default |
|----------|-------------|----------|---------|
| `DOMAIN` | Domain name for the instance (used by Caddy for TLS and LiveKit WebSocket URL) | Yes | - |
| `ACME_EMAIL` | Email address for Let's Encrypt certificate registration | Yes | - |
| `POSTGRES_PASSWORD` | PostgreSQL password for the `redoubt` database user | Yes | Generated by installer |
| `JWT_SECRET` | JWT signing secret (base64-encoded, 64 bytes) | Yes | Generated by installer |
| `STORAGE_MASTER_KEY` | Encryption key for file uploads (base64-encoded, 32 bytes) | Yes | Generated by installer |
| `LIVEKIT_API_KEY` | LiveKit API key (shared between the API server and LiveKit) | Yes | Generated by installer |
| `LIVEKIT_API_SECRET` | LiveKit API secret (base64-encoded, 32 bytes) | Yes | Generated by installer |
| `ADMIN_SESSION_SECRET` | Admin panel session signing secret (base64-encoded, 32 bytes) | Yes | Generated by installer |
| `RESEND_API_KEY` | Resend email service API key (enables email verification and password reset) | No | - |
| `S3_ACCESS_KEY` | S3-compatible storage access key (enables file uploads) | No | - |
| `S3_SECRET_KEY` | S3-compatible storage secret key | No | - |
| `GRAFANA_PASSWORD` | Grafana admin password (only used with the monitoring profile) | No | `admin` |

### Generating Secrets Manually

If you need to regenerate any secret:

```bash
# 64-byte base64 secret (for JWT_SECRET)
openssl rand -base64 64 | tr -d '\n'

# 32-byte base64 secret (for STORAGE_MASTER_KEY, LIVEKIT_API_SECRET, ADMIN_SESSION_SECRET)
openssl rand -base64 32 | tr -d '\n'

# Hex string (for POSTGRES_PASSWORD)
openssl rand -hex 32
```

After changing any value in `.env`, restart the affected services:

```bash
cd /opt/redoubt
docker compose up -d
```

## Config File (config/config.yaml)

The main application configuration file. Environment variables referenced with `${VAR_NAME}` syntax are interpolated at startup. The full annotated configuration:

```yaml
# Server settings
server:
  host: "0.0.0.0"              # Listen address
  port: 8080                    # HTTP port (behind Caddy reverse proxy)
  read_timeout: 30s             # Maximum duration for reading request body
  write_timeout: 30s            # Maximum duration for writing response
  shutdown_timeout: 30s         # Graceful shutdown timeout

# PostgreSQL database
database:
  host: "postgres"              # Hostname (Docker service name)
  port: 5432                    # PostgreSQL port
  name: "redoubt"               # Database name
  user: "redoubt"               # Database user
  password: "${POSTGRES_PASSWORD}"  # From .env
  max_open_conns: 25            # Maximum open connections
  max_idle_conns: 5             # Maximum idle connections
  conn_max_lifetime: 5m         # Maximum connection lifetime

# Redis (used for sessions, presence, pub/sub, and LiveKit coordination)
redis:
  host: "redis"                 # Hostname (Docker service name)
  port: 6379                    # Redis port
  password: ""                  # Redis password (empty for default config)
  db: 0                         # Redis database number

# Authentication
auth:
  jwt_secret: "${JWT_SECRET}"       # JWT signing key (from .env)
  jwt_expiry: 15m                   # Access token lifetime
  refresh_expiry: 720h              # Refresh token lifetime (30 days)
  password_min_length: 12           # Minimum password length
  lockout_threshold: 5              # Failed login attempts before lockout
  lockout_duration: 15m             # Account lockout duration

# Email (transactional)
email:
  provider: "resend"                # Email provider ("resend" supported)
  api_key: "${RESEND_API_KEY}"      # Provider API key (from .env)
  from_address: "noreply@example.com"  # Sender email address
  from_name: "Redoubt"             # Sender display name
  verification_expiry: 24h         # Email verification link lifetime
  reset_expiry: 1h                 # Password reset link lifetime

# File storage (S3-compatible)
storage:
  endpoint: "http://localstack:4566"  # S3 endpoint URL
  bucket: "redoubt-media"             # Bucket name
  region: "us-east-1"                 # AWS region
  access_key: "${S3_ACCESS_KEY}"      # Access key (from .env)
  secret_key: "${S3_SECRET_KEY}"      # Secret key (from .env)
  encryption:
    master_key: "${STORAGE_MASTER_KEY}"  # File encryption key (from .env)

# CORS (Cross-Origin Resource Sharing)
cors:
  allowed_origins:
    - "https://app.example.com"     # Client application origin
  allowed_methods:
    - GET
    - POST
    - PUT
    - PATCH
    - DELETE
  allowed_headers:
    - Authorization
    - Content-Type
    - X-Request-ID
  max_age: 86400                    # Preflight cache duration in seconds

# Logging
logging:
  level: "info"                     # Log level: debug, info, warn, error
  format: "json"                    # Log format: json, text

# Telemetry (OpenTelemetry)
telemetry:
  service_name: "redoubt-api"       # Service name in traces/metrics
  service_version: "${VERSION:-0.1.0}"  # Reported version
  tracing:
    enabled: true                   # Enable distributed tracing
    sample_rate: 0.1                # Trace sampling rate (0.0 to 1.0)
    otlp_endpoint: "lgtm:4317"     # OTLP gRPC collector endpoint
  metrics:
    enabled: true                   # Enable Prometheus metrics
    port: 9090                      # Metrics endpoint port (/metrics)

# LiveKit (voice and video)
livekit:
  host: "http://livekit:7880"       # LiveKit server URL (internal)
  api_key: "${LIVEKIT_API_KEY}"     # API key (must match LiveKit's LIVEKIT_KEYS)
  api_secret: "${LIVEKIT_API_SECRET}"  # API secret (must match LiveKit's LIVEKIT_KEYS)
  ws_url: "wss://${DOMAIN}/livekit"    # Client-facing WebSocket URL
  webhook_path: "/api/v1/livekit/webhook"  # Webhook callback path

# WebSocket (real-time messaging)
websocket:
  read_buffer_size: 1024            # Read buffer size in bytes
  write_buffer_size: 1024           # Write buffer size in bytes
  ping_interval: 30s                # Ping frame interval
  pong_timeout: 60s                 # Pong response timeout
  max_message_size: 4096            # Maximum message size in bytes

# Presence (online status, typing indicators)
presence:
  typing_timeout: 5s                # Typing indicator auto-clear duration
  idle_timeout: 5m                  # Time before user is marked idle

# Voice channel settings
voice:
  default_max_participants: 25      # Default max users per voice channel

# Message settings
messages:
  edit_window: 15m                  # Time window for editing sent messages

# Admin panel
admin:
  enabled: true                     # Enable the admin panel
  port: 9091                        # Admin panel listen port (localhost only)
  session_secret: "${ADMIN_SESSION_SECRET}"  # Session signing secret (from .env)
  session_expiry: 24h               # Admin session lifetime

# Rate limiting
ratelimit:
  enabled: true                     # Enable rate limiting
  rules:
    register:                       # Account registration
      limit: 5
      window: 1h
    login:                          # Login attempts
      limit: 10
      window: 15m
    forgot_password:                # Password reset requests
      limit: 3
      window: 1h
    verify_email:                   # Email verification requests
      limit: 10
      window: 1h
    general:                        # General API requests
      limit: 100
      window: 1m
    file_upload:                    # File upload requests
      limit: 10
      window: 1h
```

### Key Configuration Notes

- **`storage.endpoint`**: For production with AWS S3, change to `https://s3.amazonaws.com`. For other S3-compatible services (MinIO, Backblaze B2, Cloudflare R2), use their respective endpoint URLs.
- **`cors.allowed_origins`**: Update this to match the origin of your Redoubt client application.
- **`telemetry.tracing.sample_rate`**: Set to `1.0` during debugging, `0.1` or lower in production to reduce overhead.
- **`auth.jwt_expiry`**: Short-lived access tokens (15m default) with long-lived refresh tokens (30 days) is the recommended pattern. Shorter access token expiry improves security but increases token refresh frequency.

## LiveKit Configuration (config/livekit.yaml)

LiveKit handles WebRTC voice and video. Key settings:

```yaml
port: 7880                        # HTTP/WebSocket signaling port

rtc:
  port_range_start: 60000         # UDP port range start for WebRTC media
  port_range_end: 60100           # UDP port range end
  tcp_port: 7881                  # TCP fallback for restrictive firewalls
  use_external_ip: false          # Set to true if behind NAT
  node_ip: 127.0.0.1             # Advertised IP (set to public IP in production)

turn:
  enabled: false                  # Enable TURN relay for NAT traversal
  # udp_port: 7882
  # tls_port: 7882
  # domain: your-domain.com

redis:
  address: redis:6379             # Redis connection (shared with Redoubt API)

room:
  empty_timeout: 0                # Destroy room when last participant leaves
  max_participants: 0             # No global limit (enforced per-channel by API)

logging:
  level: info
  json: false
```

### Production Adjustments

For production deployments, update `config/livekit.yaml`:

- Set `rtc.use_external_ip: true` to enable NAT traversal.
- Remove or update `rtc.node_ip` to the server's public IP if `use_external_ip` does not work in your environment.
- Consider enabling `turn` if users are behind restrictive corporate firewalls.

## Caddy Configuration (Caddyfile)

Caddy acts as the reverse proxy and handles automatic TLS certificate provisioning via Let's Encrypt. The `Caddyfile` routes requests to internal services:

| Route | Backend | Purpose |
|-------|---------|---------|
| `/api/v1/*` | `redoubt-api:8080` | REST API endpoints |
| `/ws` | `redoubt-api:8080` | WebSocket for real-time messaging and presence |
| `/api/v1/livekit/webhook` | `redoubt-api:8080` | LiveKit webhook callbacks |
| `/livekit/*` | `livekit:7880` | LiveKit WebRTC signaling (prefix stripped) |
| `/health`, `/health/*` | `redoubt-api:8080` | Health check endpoints |
| `*` (default) | - | Returns "Redoubt API" 200 |

The domain is read from the `DOMAIN` environment variable. Caddy automatically obtains and renews TLS certificates using the `ACME_EMAIL` address.

## Docker Compose Overrides

To customize the Docker Compose configuration without modifying the main file, create a `docker-compose.override.yml` in `/opt/redoubt/`:

```yaml
# Example: expose PostgreSQL port for external access
services:
  postgres:
    ports:
      - "5432:5432"

  # Example: set custom resource limits
  redoubt-api:
    deploy:
      resources:
        limits:
          memory: 512M
```

Docker Compose automatically merges `docker-compose.yml` with `docker-compose.override.yml` when running `docker compose up`.

## Monitoring

The production compose file includes an optional monitoring stack with Prometheus and Grafana.

### Enabling Monitoring

```bash
cd /opt/redoubt
docker compose --profile monitoring up -d
```

### Services

| Service | Address | Purpose |
|---------|---------|---------|
| Prometheus | `prometheus:9090` (internal) | Metrics collection and storage |
| Grafana | `localhost:3000` (host) | Metrics visualization and dashboards |

### Prometheus Scrape Targets

Prometheus is preconfigured to scrape metrics from:

- `redoubt-api:9090` -- API application metrics
- `livekit:7880/metrics` -- LiveKit server metrics
- `caddy:2019/metrics` -- Caddy reverse proxy metrics

### Grafana Access

Grafana is accessible at `http://your-server:3000`. The admin password defaults to `admin` unless `GRAFANA_PASSWORD` is set in `.env`. Set a strong password before exposing Grafana to the internet, or access it via SSH tunnel:

```bash
ssh -L 3000:localhost:3000 user@your-server
```

Then open [http://localhost:3000](http://localhost:3000) in your browser.
